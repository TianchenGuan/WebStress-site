# WebStress site — implementation summary

Date: 2026-05-22
Site root: `WebStress-site/site/`
Deploy target: `webstress.dev` (Vercel, Root Directory = `site/`)

## Framework

- **Vite 5** + **React 18** + **TypeScript 5.6** + **react-router-dom v6**
- **Tailwind CSS 3** for the academic-paper visual style
- **No backend, no server runtime** — fully static `dist/` shipped to the
  Vercel CDN
- Data scripts in **Python 3** (`PyYAML`) read the benchmark task / variant
  / Human-140 / assignments YAMLs and emit sanitized JSON into
  `public/data/`

The fork at `TianchenGuan/WebStress-site` is a copy of the benchmark
repo; the website lives in `site/` so the fork can still pull
benchmark updates from upstream without conflicts.

## Routes implemented

| Route                          | Page                                            |
|--------------------------------|-------------------------------------------------|
| `/`                            | Landing (hero, 6 stat cards, 4 key-result cards, paper card) |
| `/tasks`                       | Task explorer with 7 filters + search (519 tasks) |
| `/tasks/:task_id`              | Per-task detail (public instruction + paired intervention) |
| `/primitives`                  | Overview of the seven cognitive primitives    |
| `/primitives/:primitive`       | Primitive detail with filtered task list      |
| `/environments`                | Overview of the seven environments            |
| `/environments/:env`           | Environment detail with diff + primitive mix + task list |
| `/results`                     | Per-(model, primitive) heatmap-style pass rate table, failure-class table, paper figures |
| `/docs`                        | Doc hub linking to in-repo guides             |
| `/docs/setup`                  | Setup + run + evaluate + reproduce-sweep      |
| `*`                            | 404                                           |

- `tasks_index.json` — 519 entries with public-safe fields only
- `primitives.json` — 7 cards (definition + targets + counts + example)
- `environments.json` — 7 cards (description + difficulty / primitive mix)
- `results_summary.json` — headline numbers + 9 agent rows + figure refs

Generated by:

- `scripts/build_public_task_index.py` (reads `../webstress/`)
- `scripts/build_public_results_summary.py` (hardcoded from paper Table 2)

Safety-scanned by:

- `scripts/check_public_export.py`

## Safety check

`scripts/check_public_export.py` scans every JSON under `public/data/`
for:

- API keys, tokens, controller secrets, `.env` references
- Real annotator names (`Weili`, `Michael`, `Xunjian`, `Tianchen`,
  `Keagan`, `Kyle`, `Royce`, `Daisy`)
- Private absolute paths (`/home/users/`, `/Users/`, `/mnt/`,
  `/usr/project/xtmp/`)
- Raw provider responses
- Hidden evaluator predicates (`canonical_diff` / `fields:eq` JSON keys)
- Hidden target ground-truth structures (`"target": {`)
- Raw rubric free-text fields

Current result: **0 unsafe matches** across 4 generated JSON files
and 8 site files scanned. See
`artifacts/public_export_safety_report.md` for the full output.

## Vercel deployment

1. In Vercel, *Add new project* → import `TianchenGuan/WebStress-site`.
2. **Root Directory**: `site/`
3. **Framework Preset**: Vite (auto-detected)
4. **Build Command**: `npm run predeploy`
5. **Output Directory**: `dist/`
6. **Environment Variables**: none
7. **Domains** tab: add `webstress.dev` + `www.webstress.dev`,
   follow Vercel's DNS instructions.
