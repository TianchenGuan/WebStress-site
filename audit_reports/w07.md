**Addendum — Cross-cutting software observation**

The React SPA's button click handlers (thread open, appointment submit, etc.) do not reliably respond to synthetic mouse events dispatched from browser automation tools. Native click() dispatches via `element.dispatchEvent(new MouseEvent('click', {bubbles:true}))` work correctly, but the framework's `left_click` coordinate-based clicks sometimes fail to trigger React synthetic event handlers on the first attempt (possibly due to the Claude agent chat overlay capturing focus). Human users clicking directly in the browser are unaffected; this is an automation-tool interaction issue only, not a UI bug.
