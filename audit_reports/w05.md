**Key cross-cutting findings:**

1. **React input binding** (lms env): `form_input` / keyboard-type do not trigger React onChange; native value setter + synthetic events required. This affects all LMS assignment submission tasks. Agent harnesses must use the native setter pattern.

2. **Referral gate bug** (patient_portal): Three tasks (pp_cancel_reschedule, pp_care_gap_analysis) are blocked because specialty appointments require approved referrals that are never seeded. The "no messages" constraint in care_gap and absence of any remedy in reschedule make these tasks unsolvable by design.

3. **Aggressive variants**: 7 of 14 variants make the task completely unsolvable (no peer reviews, no modules, no courses, no slots, no grade data). Per the framework's own "Recoverable" design principle, a variant should be "a filter, not a wall" — these violate that principle.

4. **Eval logic bugs**: Two tasks have inverted/misfiring negative checks (lms_view_late_policy, lms_submit_peer_review, lms_three_module_chain) that penalize correct completions.
