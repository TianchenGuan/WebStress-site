## Cross-cutting issues

1. **All appointment action buttons (`Schedule New`, `Reschedule`, `Cancel`) use `type="submit"` without a wrapping `<form>` element.** AT-level clicks do not reliably fire the React onClick handler; only `dispatchEvent(new MouseEvent('click', {bubbles:true}))` works. This is a systematic SPA event-wiring bug that affects every scheduling task. It creates a false signal — the action appears to succeed visually but the underlying state is not mutated. Fix the SPA to either wrap button groups in `<form>` elements or fire onClick unconditionally on AT-level click events.

2. **Sessions created via the per-env launcher (`/env/patient_portal/launch`) are not recognised by the `/api/env/patient_portal/*` REST endpoints** (404 on all API calls). Only sessions created via the main `/api/env/patient_portal/session` POST are valid for API interaction. This is an operational inconsistency that could confuse agent harnesses that navigate to the env-local launcher.
