This is logged as 503 in the network tab on every task load but has no runtime effect since the trajectory recorder loads from `/static/trajectory-recorder.js` instead.

Degradation notes:
- variant: gmail_create_label__label_retry
- still solvable: yes
- observation: "First label creation fails transiently; agent must retry and verify" — adds one backtrack loop; reasonable test if the underlying input focus issue is fixed.

Recommend: FIX — label name input does not receive focus on click; this causes silent task failure for any agent using standard keyboard input. Fix the input's focus/blur behavior or add an explicit autofocus attribute.
