1. The Seven Cognitive Primitives

We decompose web agent competence into 7 orthogonal primitives. Each describes a distinct capability; an agent can be strong on some and weak on others. The benchmark measures each independently.

Grounding

Map observation to the correct semantic understanding of UI state. Distinguish real targets from decoys.

Without it: clicks wrong element, trusts misleading signals, confuses similar items
Test: silent_fail on API + confusing decoy data

Planning

Decompose a goal into ordered sub-goals, respecting dependencies between steps.

Without it: misordered steps, dropped constraints, monolithic attempts
Test: scrambled timestamps + stale_data on search

State Tracking

Maintain working memory of what's done vs. pending across multi-step tasks.

Without it: forgets extracted values, opens wrong item, loses count
Test: distractor emails from the same sender with similar subjects

Backtracking

Detect a dead-end or failure, then revert to a prior decision point and try an alternative.

Without it: perseveration loops, continues past errors, never retries
Test: error_then_success (503) on critical send action

Patience

Know when to wait vs. re-act. Calibrate retry timing across progressive delays.

Without it: race conditions, reads stale state, double submissions
Test: delay (progressive 1s→3s→5s) + transient 503

Exploration

Discover affordances and alternative paths when the obvious approach fails.

Without it: premature abandonment, misses hidden features
Test: silent_fail on delete + distractor noise

Verification

After performing an action, check whether it actually achieved its intended effect.

Without it: trusts fake success responses, silent failure propagation
Test: silent_fail returns fake 200 OK, data not persisted
Orthogonality: Grounding vs Verification = perceiving correctly vs inspecting outcome. Planning vs State Tracking = generating a sequence vs maintaining coherence during execution. Backtracking vs Exploration = recovering from known failure vs probing the unknown. Patience is a unique temporal dimension orthogonal to all others.

2. Why It Works — Design Rationale

Degradation operates on three independent dimensions. Each is real — not a mocked overlay — and affects the agent through the same interfaces it uses normally.

Dimension 1: Data Content (Seed & Server Layers)

Before the agent sees its first observation, the seed layer and server layer mutate the actual GmailState object that backs all API responses. These are not cosmetic overlays — they change the ground-truth data the environment serves.

  • Distractor injection (inject_distractor_emails) — adds 5 real Email objects to state.emails with the same sender and similar subjects as the target. The SPA renders them in the inbox list. The API returns them in search results. There is no metadata flag marking them as distractors — the agent must distinguish targets from noise by reading content.
  • Timestamp scrambling (scramble_timestamps) — applies random timedelta offsets to every email.timestamp. The inbox is re-sorted by the SPA, so chronological reasoning breaks. The agent must use content-based cues instead.
  • Contradictory updates (add_contradictory_update) — inserts a newer email that contradicts an older one. The agent must identify which supersedes which by date and sender authority.
  • Prerequisite removal (hide_prerequisite) — deletes a label or contact the task description assumes exists. The agent must detect the missing prerequisite and create it before proceeding.

Because these mutations happen on the canonical GmailState, every subsequent API call (list emails, search, get thread, etc.) reflects the degraded data. The SPA faithfully renders whatever the API returns, so the agent sees a consistent but harder world.

Dimension 2: Web Logic (Network Middleware Layer)

The network layer intercepts live HTTP requests inside FastAPI via DegradationMiddleware. When the React SPA calls POST /api/env/gmail/emails/X/star, the request passes through the middleware before reaching the route handler. The middleware can:

  • Return a real 503 Service Unavailable (error_then_success) — the SPA receives a genuine HTTP error. Its error handling (or lack thereof) determines what the agent sees. The server state is untouched. On the next retry the counter exceeds error_count and the request passes through normally.
  • Return a fake 200 OK (silent_fail) — the response body has the exact JSON shape the SPA expects ({"email": {"id": "...", "subject": "..."}}), so no JS error is thrown. But the route handler never ran — the server state is not mutated. The star was never set, the email was never sent. The agent can only discover this by navigating to the Sent folder or re-checking the starred status.
  • Inject real latency (delay) — await asyncio.sleep(delay_ms / 1000) blocks the response. Playwright's timeout and the browser's loading indicator reflect the wait. Progressive mode escalates: 1s for the first 3 calls, 3s for the next 5, 5s thereafter.
  • Serve stale data (stale_data) — the first N GET requests return a cached (empty or outdated) response body. The agent sees an empty search result or an old inbox snapshot, and must re-query.

This layer is the core innovation. Because it operates at the HTTP level inside the server process, it works identically for human browsers and Playwright agents — same 503, same fake 200, same latency. No Playwright-specific hooks, no DOM timing races.

Dimension 3: UI Presentation (Client Layer)

The client layer mutates the rendered DOM via JavaScript. When a session has client degradation registered, the benchmark-toolbar.js script fetches the injection config from GET /api/env/gmail/degradation/{session_id} and applies mutations:

  • ARIA label scrambling (scramble_aria) — shifts aria-label attributes between sibling elements. Agents relying on accessibility semantics for element identification are misdirected.
  • Affordance hiding (hide_affordance) — sets display:none on a button and reveals it on a non-obvious trigger (e.g., right-click). Agents must explore alternative interactions.
  • False banners (false_banner) — injects a misleading alert element (e.g., "Feature temporarily unavailable"). Grounded agents check actual page behavior instead of trusting the banner.

Persistent mode uses a MutationObserver to re-apply mutations after SPA route changes. Caveat: client-layer injections have a race condition with BrowserGym's AXTree snapshots (the observer fires after the snapshot). For reliable agent testing, we prefer network-layer alternatives (e.g., silent_fail instead of hide_affordance).

Why All Three Dimensions Together

Each dimension alone is insufficient. Data content changes test whether the agent can find the right information in a noisy environment. Web logic changes test whether the agent can handle failures and verify outcomes. UI presentation changes test whether the agent can navigate despite misleading signals.

Combined in a single variant, they create realistic stress conditions: the inbox is noisy (seed), some API calls fail silently (network), and navigation labels are misleading (client). An agent needs the target primitive to overcome all three at once — this is why the degradation is a filter (solvable with the right capability) rather than a wall (impossible regardless).

Evaluation ground truth: Check expressions run against the live server state (not DOM snapshots). A silently-failed star or send is caught regardless of what the UI shows, because the evaluator asks state.get_email('email_3').is_starred — and the answer is False if the middleware intercepted the request.

7. Network Middleware Deep Dive

injector/middleware.py — The core innovation. All degradation is server-side, so it works identically for human browsers and Playwright agents.

class DegradationMiddleware(BaseHTTPMiddleware):

    async def dispatch(self, request, call_next):
        # ── Step 1: Identify session (without reading body) ──
        session_id = (
            request.query_params.get("session_id")           # GET ?session_id=X
            or _extract_session_from_referer(referer)         # POST Referer: ?session=X
            or _extract_session_from_cookie(cookies)          # wab_session=X cookie
        )
        # Fallback: single-session heuristic for agent_eval runs
        if not session_id and len(_SESSION_NETWORK) == 1:
            if request.method in ("POST", "PUT") and "/api/env/" in url_path:
                session_id = next(iter(_SESSION_NETWORK))

        # ── Step 2: Match URL pattern ──
        for inj in _SESSION_NETWORK[session_id]:
            if _url_matches_pattern(url, inj["url_pattern"]):

                # ── Step 3: Apply degradation action ──
                if action == "error_then_success":
                    call_num = _get_counter(session_id, key)
                    if call_num <= error_count:
                        return JSONResponse(status_code=503, ...)  # real 503!

                elif action == "silent_fail":
                    if call_num <= fail_count:
                        return JSONResponse(200, body=fake_success)  # looks OK, data lost

                elif action == "delay":
                    await asyncio.sleep(delay_ms / 1000)          # real latency

        return await call_next(request)  # normal handler

Deterministic Firing (Seeded)

def _seeded_should_fire(seed: int, call_index: int, probability: float) -> bool:
    """md5(seed:call_index) → deterministic coin flip. Same seed = same pattern."""
    h = hashlib.md5(f"{seed}:{call_index}".encode()).hexdigest()
    return (int(h[:8], 16) / 0xFFFFFFFF) < probability

URL Pattern Matching

# Pattern: "**/api/env/gmail/emails/*/star"
# URL:     "http://127.0.0.1:8080/api/env/gmail/emails/email_3/star"
# ** strips prefix, * matches one path segment → match!
def _url_matches_pattern(url, pattern):
    if pattern.startswith("**/"):
        regex = _glob_to_regex(pattern[3:])
        return bool(re.search(regex, url))

8. Session Creation & Injection Registration

backend/routes/gmail.py create_session() — the critical path where degradation is applied.

@router.post("/session")
def create_session(body: SessionCreateRequest, session_manager):
    # 1. Validate task/variant binding
    if degradation.base_task_id != body.task_id:
        raise HTTPException(400, "variant bound to wrong task")

    # 2. Create seeded session (deterministic GmailState)
    session_id, resolved_targets, seed = session_manager.create_session(
        "gmail", body.task_id, body.seed)

    # 3. Store degradation metadata on session state
    state._degradation = {variant_id, base_task_id, target_primitive, injections}

    # 4. Apply seed + server injections (mutate state in-place)
    for inj in degradation["injections"]:
        if inj.layer == "seed":
            seed.apply_seed_injection(state, inj.params)
        elif inj.layer == "server":
            server.apply_server_injection(state, inj.params)

    # 5. Register network + client for middleware interception
    middleware.register_session_degradation(session_id, injections)
    #    → _SESSION_NETWORK[sid] = [network injections]
    #    → _SESSION_CLIENT[sid]  = [client injections]

    return {session_id, resolved_targets, instruction, degradation_active: True}

9. Expression-Based Evaluator

tasks/_evaluator.py — Check expressions run against live GmailState in a restricted namespace.

def evaluate(task, *, server_state, targets, trajectory):
    # Restricted namespace — no __import__, no file I/O
    namespace = {
        "__builtins__": {any, all, len, str, int, float, sorted, min, max, sum, abs},
        "state": server_state,      # the live GmailState
        "target": _DotDict(targets), # {target.email_id} → "email_3"
    }

    # Evaluate positive checks
    for check in task.eval.checks:
        expr = _substitute_targets(check.expr, targets)  # "{target.X}" → "email_3"
        passed = eval(compile(expr, "<check>", "eval"), namespace)

    # Evaluate negative checks (penalties, capped at 0.95)
    penalty = min(sum_of_failed_penalties, 0.95)

    score = (passed_count / total_checks) - penalty      # clamped to [-1, 1]
    success = all_positive_passed and score >= 0.5

10. BrowserGym Integration

browsergym_task.py implements AbstractBrowserTask from BrowserGym.

class WebAgentBenchTask(AbstractBrowserTask):

    def setup(self, page) -> (str, dict):
        # 1. Start server if not running
        self._ensure_server()
        # 2. POST /session with full degradation config
        session = _http_json(url, method="POST", payload={
            task_id, seed, degradation: {variant_id, injections: [...]}
        })
        # 3. Navigate Playwright page
        page.goto(f"/env/gmail/inbox?session={sid}&agent_mode=1")
        #    agent_mode=1 suppresses human toolbar, keeps degradation active
        return (instruction, task_info)

    def validate(self, page, chat_messages) -> (reward, done, msg, info):
        # Detect agent calling send_msg_to_user() (new msgs beyond initial greeting)
        new_messages = chat_messages[self._initial_chat_count:]
        if any(m.role == "assistant" for m in new_messages):
            result = _http_json("/evaluate", method="POST", payload={session_id, task_id})
            return (result["score"], True, "", {"evaluation": result})
        return (0.0, False, "", {})  # not done yet

    def teardown(self):
        _http_json(f"/session/{sid}", method="DELETE")  # also unregisters degradation
agent_mode=1 in the URL query prevents benchmark-toolbar.js from rendering the human evaluation toolbar. The client degradation fetch (GET /degradation/{sid}) still runs, so DOM mutations apply to agents too.

11. Agent Observation Pipeline

agent.py — Converts BrowserGym's raw observation dict into LLM-friendly text.

def format_obs_for_llm(obs: dict) -> str:
    from browsergym.utils.obs import flatten_axtree_to_str

    parts = []
    parts.append(f"Task: {obs['goal']}")
    if obs["last_action_error"]:
        parts.append(f"Last action error: {obs['last_action_error']}")

    # Use BrowserGym's native AXTree formatter (not raw CDP dicts!)
    axtree_txt = flatten_axtree_to_str(
        obs["axtree_object"],
        extra_properties=obs.get("extra_element_properties", {}),
        with_clickable=True, with_visible=True,
        filter_visible_only=True, filter_with_bid_only=True,
    )
    parts.append(f"Accessibility tree:\n{axtree_txt}")
    return "\n".join(parts)

# injector/config.py
@dataclass
class Injection:
    layer: Literal["client","network","server","seed"]
    params: dict

@dataclass
class DegradationConfig:
    variant_id: str
    base_task_id: str
    target_primitive: str
    injections: list[Injection]

    @classmethod
    def from_yaml(cls, path):
        raw = yaml.safe_load(open(path))
        injections = [Injection(**inj)
                      for inj in raw.pop("injections")]
        return cls(**raw, injections=injections)

13. Delta Computation

scripts/compute_delta.py — Compares standard vs degraded result JSONs.

# Usage:
# python -m webagentbench.scripts.compute_delta \
#     --standard results/normal.json \
#     --degraded results/stress_patience.json results/stress_grounding.json

def compute_delta(standard_path, degraded_paths) -> dict:
    standard_scores = {r.task_id: r.score for r in standard_results}
    standard_times  = {r.task_id: r.elapsed for r in standard_results}

    for deg_result in all_degraded_results:
        delta = standard_scores[tid] - deg_result.score
        time_ratio = deg_result.elapsed / standard_times[tid]

        # Signal detection:
        if delta > 0.1:       signal = "WEAK"    # score dropped
        elif time_ratio > 1.3: signal = "ADAPT"   # score held, agent adapted
        else:                  signal = "-"       # no effect

14. Key Engineering Decisions

DecisionWhyFiles
Server-side middleware for ALL network degradation Client-side (Playwright page.route()) has race conditions with BrowserGym AXTree snapshots. Server-side intercepts work identically for humans and agents. injector/middleware.py
Single-session fallback for POST session extraction React Router drops ?session= from Referer on SPA navigation. POST bodies can't be read without breaking Pydantic parsing. When exactly 1 session is active (agent_eval mode), assume it. middleware.py:dispatch()
Guaranteed first-N-fail, not probabilistic 20% probability × 3 API calls = 51% chance of zero hits. Use error_count: 2 (mode: once) for reliable testing. variants/*.yaml
BrowserGym's flatten_axtree_to_str() for observation Raw CDP AXTree dicts are unreadable by LLMs. BrowserGym's formatter produces [75] button 'Star...' format with bid references. agent.py:format_obs_for_llm
Track _initial_chat_count in validate() BrowserGym seeds 2 chat messages (greeting + goal) on reset. Without tracking, validate() terminates every episode at step 1. browsergym_task.py:validate
Expression evaluator against server state (not DOM) Server state is deterministic and fast. DOM checks require browser interaction and are fragile to layout changes. All Gmail mutations go through the API, so server state is ground truth. tasks/_evaluator.py
Time-ratio as secondary metric Patience/planning degradation may not drop scores (agent adapts) but increases time 1.6–2.1x. Score-only delta misses this signal. scripts/compute_delta.py
